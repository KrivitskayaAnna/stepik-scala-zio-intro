# Проект 1

## Условие задания

В данном проекте вам необходимо реализовать программу, которая запускается по расписанию,
проверяет и "скачивает" новые события из внешнего API, а затем "сохраняет" их в базу данных 
(вместо реальных внешнего API и базы используйте заглушки-методы из классов clients/)

Алгоритм работы программы:

1. Программа стоит на расписании - запускается раз в 10 минут<br/>
   Для постановки эффекта на расписание используйте effect.repeat(zio.Schedule...).<br/>
   Если предыдущий запуск программы не успел доработать к моменту, когда пора делать следующий запуск, то программа<br/>
   должна дождаться окончания предыдущего запуска и сразу запуститься снова.<br/>
2. При запуске программа логирует инфо о начале обработки<br/>
   Для логирования используйте ZIO.logInfo().<br/>
3. Программа получает авторизационный токен из API<br/>
   Используйте метод getSessionToken из ApiClient.<br/>
   Если токен не удалось получить, то программа логирует ошибку "Не удалось получить токен" и текст ошибки от API.<br/>
   Для логирования ошибки используйте ZIO.logErrorCause().<br/>
   Затем программа не падает, а просто ждет до следующего запуска по расписанию.<br/>
   Для обработки и перехвата ошибки используйте effect.foldCauseZIO(...).<br/>
4. Программа достает из базы идентификатор последнего скачанного события<br/>
   Используйте метод getLastEventId из DatabaseClient.<br/>
   Ошибки в запросах к базе считаем фатальными, то есть при такой ошибке приложение должно падать.<br/>
5. Программа обращается на API с идентификатором последнего события и получает по нему идентификаторы более свежих
   событий<br/>
   Используйте метод getEventsNewerThan из ApiClient.<br/>
   Если более свежих событий нет, то программа логирует эту инфу и просто ждет следующего запуска.<br/>
   Если более свежие события есть, то программа логирует их количество и идентификаторы и продолжает для них обработку.<br/>
6. Для каждого свежего события программа обращается на API за вложением с подробной информацией по событию<br/>
   Используйте метод getEventAttachment из ApiClient.<br/>
   Для итерации по всем событиям используйте метод ZIO.foreach(aList)(listElementEffect).<br/>
   Для оптимальности добавьте параллелизм к вызовам API, однако количество одновременных запросов на API
   не должно превышать n (например, 3).<br/>
   Если для каких-то событий не удалось получить вложения из-за ошибок, то программа логирует эту инфу вместе с
   идентификаторами таких событий и идет дальше.<br/>
   Для разбиения списка на успешные и неуспешные обработки используйте ZIO.partition(aList)(listElementEffect).<br/>
7. Программа отфильтровывает события по типам<br/>
   Информация о типе события находится в поле eventType из вложения.<br/>
   Программа выбирает все события, кроме событий типа 'fraudCall', а также логирует инфо о количестве и идентификаторах
   отфильтрованных и оставшихся событий.<br/>
   Для фильтрации используйте параллельную весию ZIO.filterNot(aList)(zioPredicate).<br/>
8. Программа сохраняет выбранные события в базу<br/>
   Используйте метод batchSaveFilteredEvents из DatabaseClient.<br/>
9. Программа логирует инфо об окончании процессинга и ждет следующего запуска<br/>
   Необходимо залогировать максимальный успешно обработанный идентификатор события, количество событий, а также время в
   секундах, сколько всего заняла вся обработка от момента запуска до момента окончания.<br/>
